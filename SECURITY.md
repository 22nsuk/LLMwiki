# Security Policy

## Supported Surface

현재는 기본 브랜치의 최신 상태만 지원한다.
공개 mirror 기준으로는 code/ops runtime과 테스트 surface를 지원 범위로 본다.

## Reporting a Vulnerability

보안 문제로 보이는 이슈는 공개 issue나 공개 PR로 먼저 올리지 않는 편이 좋다.

권장 순서:

1. GitHub의 Private Vulnerability Reporting이 활성화되어 있으면 그 경로를 사용한다.
2. 그 경로가 없다면, 공개 채널 대신 현재 프로젝트의 private maintainer contact 경로로 먼저 알린다.
3. 재현 절차, 영향 범위, 가능한 완화책을 함께 적어준다.

## What to Include

- 영향을 받는 경로 또는 기능
- 재현 단계
- 예상 영향
- 가능한 경우 최소 재현 예시

## Response Goal

- 접수 확인
- 영향도 분류
- 수정 또는 완화 계획 공유

이 문서는 public mirror 기준의 기본 정책이다.
저장소 owner가 구체적인 연락 채널을 확정하면 이 파일을 그 값으로 갱신하는 편이 좋다.
