# Security Policy

## Supported Surface

Only the latest state of the default branch is supported. For the public mirror,
the supported surface is the code/ops runtime and test surface.

## Reporting a Vulnerability

Do not post suspected vulnerabilities in a public issue or public pull request
before a maintainer has had a chance to triage them privately.

Use this order:

1. Use GitHub Private Vulnerability Reporting for this repository when it is
   available from the repository Security tab:
   <https://github.com/22nsuk/LLMwiki/security/advisories/new>.
2. If Private Vulnerability Reporting is unavailable, use the repository
   owner's GitHub-listed private contact channel. If no private contact is
   listed, open a minimal public issue requesting a private security contact,
   but do not include exploit details, secrets, or affected private paths.
3. Include reproduction steps, impact, affected public paths or features, and
   any known mitigation.

## What to Include

- Affected path or feature
- Reproduction steps
- Expected impact
- Minimal reproduction, when possible
- Known mitigations or workarounds

## Response Goal

- Acknowledge receipt
- Classify impact
- Share a fix or mitigation plan

This is the baseline policy for the public mirror. If the repository owner
publishes a more specific private security contact, update this file with that
channel.
